# 10. ainer-worker-llm

## 定位
LLM 执行器（抽取/翻译/提示词/改写）。

## 核心职责
- 执行文本类任务并返回结构化结果。
- 记录 token、延迟、成本与 provider 选择。
- 支持多 provider 回退链。

## 输入
- task payload、RAG 片段、策略约束。

## 输出
- `job.succeeded/job.failed`（主状态）、`worker.llm.completed`（执行明细）、`worker_result`、`cost_ledger`。

## 上下游
- 上游：`07.ainer-worker-hub.md`。
- 下游：`03/05/10 skill` 逻辑消费层与 `13 observer`。

## 边界
- 不写编排终态。
- 不负责模型路由决策（由 06 负责）。

## 失败策略
- provider 失败：按回退链切换。
- 响应不合法：标记 malformed 并请求重试或人工审核。
